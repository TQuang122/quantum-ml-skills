# qml-paper-replication example workflows

## Example 1 — extract a paper into a build plan

- Input: “Implement the variational classifier described in this QML paper.”
- Skill outcome: extracts explicit assumptions, identifies missing details, defines baselines, and produces an implementation checklist.

## Example 2 — document deviations honestly

- Input: “The paper does not specify the exact split seed or backend details.”
- Skill outcome: records those gaps as explicit assumptions instead of silently guessing.

## Example 3 — classify the result correctly

- Input: “My results follow the same trend as the paper, but the final metric is weaker.”
- Skill outcome: classifies the outcome as `approximated` rather than overclaiming full replication.
